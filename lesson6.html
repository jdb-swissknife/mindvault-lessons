Lesson 6
Agent Safety Basics
Use powerful agents with clear boundaries

The Problem With Trust

As your agent gets more useful, a natural question shows up: what should I trust it with?

That question is healthy. You should not blindly hand a powerful system every password, client detail, and business decision.

But fear can create the opposite problem too. If you never give the agent useful context, it cannot do useful work.

The goal is not blind trust. The goal is clear boundaries.

The Solution

Today you are creating your agent safety rules card. These rules help you use agentic AI with confidence.

  1. Know what not to share -- Passwords, private secrets, sensitive client data
  2. Use safe access paths -- Config files, official login flows, and permissioned tools
  3. Confirm before risky actions -- Money, deletion, publishing, sending, legal, hiring, firing
  4. Ask for the plan first -- Make the agent explain what it will do
  5. Keep judgment with you -- The agent supports decisions. It does not own them
Step 1

Save Your Safety Rules

Copy this into your agent so the boundaries are clear.

Remember these safety rules for working with me: 1. Never ask me to type passwords in chat. 2. If credentials are needed, give me a safe config path, official login flow, or manual setup step. 3. Before deleting, sending, publishing, charging, buying, or changing access, explain the plan and ask for confirmation. 4. Do not make legal, financial, hiring, firing, medical, or client-commitment decisions for me. Help me think, but keep me in charge. 5. If something is sensitive, tell me what you need and the safest way to provide it. When I say "safety check," review the current task against these rules.

Step 3

Ask for a Plan Before Action

When an agent is about to do something with consequences, make it slow down.

Before you take action, give me a safety check. Tell me: 1. What you are about to do 2. What could go wrong 3. What information or access you need 4. Whether this changes, deletes, sends, publishes, buys, or commits anything 5. What you need me to confirm before proceeding

This is not about making the agent timid. It is about making the work controlled.

Step 4

Know the Human-in-the-Loop Moments

Keep yourself in the loop for decisions that carry real consequences:

  • Sending messages to customers
  • Publishing public content
  • Deleting files or records
  • Charging money or changing billing
  • Signing contracts or making legal claims
  • Hiring, firing, discipline, or employee issues
  • Anything involving private client information

What's Happening Behind the Scenes

You are learning the difference between capability and permission.

CapabilityPermission
The agent can draft an emailYou decide when it gets sent
The agent can inspect filesYou decide what files it can access
The agent can recommend a decisionYou make the decision
The agent can run a workflowYou set the boundaries

The Bigger Picture

Strong systems need strong boundaries. This lesson gives you the confidence to keep learning without treating every new capability like a risk.

The real takeaway: Safe agent use is not about doing less. It is about making the right things explicit before the agent acts.
