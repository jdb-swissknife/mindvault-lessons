Lesson 5
Tool Request Builder
Describe the next workflow you want your agent to handle

The Problem With "Can AI Help With This?"

Most business owners can feel where the friction is. Follow-ups. Notes. Estimates. Hiring. Scheduling. Reporting. But they do not always know how to turn that friction into an agent task.

So they ask vague questions:

  • "Can you automate my sales?"
  • "Can AI help with operations?"
  • "Can we make a tool for this?"

Those are fair questions, but they are too broad. Agents work best when you describe the workflow.

The Solution

Today you are learning how to write a clear tool request. This is how non-technical users start shaping agentic systems.

  1. Name the repeated work -- What do you do again and again?
  2. List the inputs -- What information does the agent need?
  3. Define the output -- What should come back?
  4. Set the trigger -- When should it happen?
  5. Describe success -- What does done right look like?
The shift: Stop asking for generic AI help. Start describing repeatable business workflows.
Step 1

Use the Tool Request Formula

Copy this prompt into your agent. It will interview you and turn your answers into a clean request.

I want to design a new agent tool or workflow. Interview me using these five questions, one at a time: 1. What do I do repeatedly? 2. What information does the agent need? 3. What should the agent produce? 4. When should this happen? 5. What does "done right" look like? After I answer, write a clean tool request I can save or send to MindVault.

This teaches your agent to slow you down and get the important details before jumping to a solution.

Step 3

Build Your First Request

Use this if you already know the workflow you want to improve.

Turn this into a clear agent tool request. Repeated work: [what you do again and again] Inputs: [what information the agent needs] Output: [what the agent should produce] Trigger: [when this should happen] Done right means: [what a good result looks like] Make the request simple enough that a non-technical business owner can understand it.

Your Assignment

Your to-do list:

The Bigger Picture

This is where your agent starts feeling less like a chat window and more like an operations designer.

You do not need to know how to code the system. You need to know how to describe the work clearly.

The real takeaway: If you can describe the workflow, your agent can help you shape the tool.
